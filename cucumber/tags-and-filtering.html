Tags & Filtering

Organize and selectively run scenarios using tags — @smoke, @regression, @wip, tag expressions, and tagged hooks.

Tags — Organizing & Filtering Tests

Tags let you categorize scenarios and selectively run subsets of your test suite. Tag a scenario as @smoke for quick health checks or @wip to exclude work-in-progress. Tag expressions support and, or, and not logic.

tagged_features.feature 
@regression
Feature: Shopping Cart

  @smoke @priority-high
  Scenario: Add item to empty cart
    Given the cart is empty
    When the user adds "Laptop" to the cart
    Then the cart should contain 1 item

  @smoke
  Scenario: Remove item from cart
    Given the cart contains "Laptop"
    When the user removes "Laptop" from the cart
    Then the cart should be empty

  @slow @integration
  Scenario: Apply discount coupon
    Given the cart contains items worth "$500"
    When the user applies coupon "SAVE20"
    Then the total should be "$400"

  @wip
  Scenario: Gift wrapping option
    Given the cart contains "Watch"
    When the user selects gift wrapping
    Then gift wrapping fee should be added
TagExamples.txt 
# Tag expression examples for runner or CLI

# Run only @smoke scenarios
@CucumberOptions(tags = "@smoke")
mvn test -Dcucumber.filter.tags="@smoke"

# Run @smoke AND @priority-high
@CucumberOptions(tags = "@smoke and @priority-high")

# Run @smoke OR @regression
@CucumberOptions(tags = "@smoke or @regression")

# Exclude @wip scenarios
@CucumberOptions(tags = "not @wip")

# Complex expression
@CucumberOptions(tags = "(@smoke or @regression) and not @wip")

# Run everything except @slow
@CucumberOptions(tags = "not @slow")

# Feature-level tags are inherited by all scenarios
# @regression on Feature means all scenarios get @regression

tag_commands.sh 
# Behave tag filtering commands

# Run only @smoke scenarios
behave --tags=@smoke

# Run @smoke AND @priority-high
behave --tags="@smoke and @priority-high"

# Run @smoke OR @regression
behave --tags="@smoke,@regression"

# Exclude @wip
behave --tags="not @wip"

# Complex: smoke or regression but not wip
behave --tags="(@smoke or @regression) and not @wip"

# Multiple --tags are ANDed
behave --tags=@smoke --tags="not @slow"

# Set default tags in behave.ini:
# [behave]
# default_tags = not @wip
